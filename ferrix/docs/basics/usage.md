# Использование FSM

После того, как вы установили FSM, его запуск осуществляется командой:

```bash
ferrix-app
```

Обратите внимание на то, что запуск должен производиться от имени *обычного*, непривилегированного пользователя. Запуск программы от имени пользователя `root` заблокирован и это поведение стандартными средствами изменить нельзя. Все действия, для которых нужен `root`, выполняются через "посредника" --- `ferrix-polkit`, который запускается программами `pkexec` или, в случае его отсутствия, `gksudo`.

После запуска вам откроется окно дашборда (картинка кликабельна):

<a href="https://mskrasnov.github.io/ferrix/assets/dashboard.png"><img src="https://mskrasnov.github.io/ferrix/assets/dashboard.png" width="60%"></a>

Основная идея дашборда заключается в оценке состояния системы *в данный момент времени*, поэтому тут нет графиков или чего-то в этом роде. Каждый компонент системы, существующий в дашборде, отображается в своей карточке, которая сверху имеет заголовок, а снизу область данных. По нажатию на карточку откроется соответствующая ей страница с более расширенной информации по конкретному просматриваемому компоненту.

Содержимое дашборда от системе к системе может отличаться. Например, если у вас нет подкачки, то карточка "Swap" не будет отображена; с другой стороны, если у вас несколько файлов/разделов подкачки, то у вас будет отображено несколько карточек "Swap". Тоже самое относится и к карточкам "Battery", "Root/Home Partition".

В целом, UX программы похож на UX классических представителей жанра --- Hardinfo или AIDA64: слева расположен сайдбар с кнопками выбора страниц с данными по конкретному компоненту ПК/ОС, справа -- область отображения этих данных в табличном/графическом виде.

<a href="https://mskrasnov.github.io/ferrix/assets/sysmon.png"><img src="https://mskrasnov.github.io/ferrix/assets/sysmon.png" width="60%"></a>

Следующая страница -- "Системный монитор" ("System Monitor"). В отличие от дашборда, системный монитор предназначен уже для просмотра изменения использования ЦП, ОЗУ во времени. На эту страницу можно перейти как из сайдбара, так и нажав по карточке "CPU Usage" ("Нагрузка на ЦП"). Только если карточка "CPU Usage" показывает *общую* нагрузку на все ядра ЦП, то на графике будут отображены сведения о нагрузке на каждое ядро ЦП.
